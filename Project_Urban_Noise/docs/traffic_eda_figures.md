# Traffic Volume EDA — 图表说明文档

**数据来源**：NYC Automated Traffic Volume Counts（自动计数，`Automated_Traffic_Volume_Counts_20260209.csv`）和 Historical Traffic Volume Counts（历史计数，`Traffic_Volume_Counts_(Historical)_20260209.csv`），经过空间匹配（交通传感器 SegmentID → block group）和月度聚合后，生成面板数据 `data/processed/monthly_dynamic.parquet`。

**面板数据规模**：560 万行，28,790 个 block group，时间跨度 2010 年 1 月 — 2026 年 3 月。

**核心字段**：

| 字段 | 含义 |
|------|------|
| `bgrp_id` | Block group 唯一标识（格式：Borough_Block，如 `BX_1234`） |
| `month` | 月份（每月第一天，格式 `YYYY-MM-01`） |
| `traffic_volume_sum` | 该 bgrp 当月累计交通流量（来自自动计数传感器） |
| `traffic_obs_count` | 当月观测记录条数 |
| `traffic_hist_daily_sum` | 历史计数数据的日均加总 |

## Figure 01 — 数据覆盖概览

**文件**：`figures/traffic_eda/01_coverage_heatmap.png`

**生成数据**：`monthly_dynamic.parquet` 全量面板，对 `traffic_volume_sum > 0` 的记录按年和 bgrp 分组统计。

**左图：每年有数据的 bgrp 数量**

横轴为年份，左纵轴（蓝柱）为当年至少有一个月记录不为零的 bgrp 数量，右纵轴（红线）为其占全部 28,790 个 bgrp 的百分比。

**右图：每个 bgrp 累计有多少个月有数据**

横轴为"有效月份数"，纵轴为 bgrp 数量。中位数为 1 个月，均值约 2 个月。全部 28,790 个 bgrp 中仅 1,844 个（6.4%）曾出现过不为零的记录。

**揭示的问题**：

- 数据极度稀疏。绝大多数 bgrp 在整段时间窗口内只有 1–2 个月有实际观测，而非连续时间序列。
- 传感器覆盖率在 2010–2014 年相对最高（约 0.8%），此后持续下降，近年仅约 0.2–0.3%，意味着近年数据反而更稀缺。
- **建模影响**：traffic 不是一个对所有 bgrp 都均匀可用的特征；直接使用零填充会掩盖"无传感器"和"真实零流量"的区别，需要谨慎处理缺失机制。

## Figure 02 — 全市月度时间序列

**文件**：`figures/traffic_eda/02_time_series.png`

**生成数据**：全量面板按月聚合，计算三条指标：全市总流量、每 bgrp 平均流量、当月有不为零记录的 bgrp 数量。

**三个子图（共用横轴：月份）**：

- **红线（上）**：全市总流量，单位为次/月。整体在 0 到约 200 万之间波动，呈现高度不规则的锯齿形，存在多个显著尖峰。
- **蓝线（中）**：每 bgrp 平均流量，走势与总量相似，量级在 0–70 之间，反映单个 block group 的平均负荷。
- **绿线（下）**：当月有数据的 bgrp 数量，大多数月份在 20–40 之间，偶有峰值超过 70，但整体极低（相对于全市 28,790 个 bgrp）。

**揭示的问题**：

- 时间序列高度不平稳，存在剧烈波动，不像通常认知中"交通量随季节平滑变化"的形态。
- 锯齿状剧烈波动很可能不是真实交通波动，而是**传感器上报批次不均匀**造成的——某些月份集中上报了更多传感器的历史数据，导致统计值陡然上升。
- 没有出现明显的 2020 年 COVID 低谷，进一步印证数据覆盖本身就断断续续，单月的绝对值受上报传感器数量影响远大于真实交通变化。
- **建模影响**：直接使用月度总量或均值作为特征会引入数据采集噪声，建议使用经过传感器数量归一化后的指标，或改用二值覆盖变量。

## Figure 03 — 月度平均（年内季节模式）

**文件**：`figures/traffic_eda/03_monthly_average.png`

**生成数据**：剔除零值后，对全量面板按日历月份（1–12 月）分组，计算均值和中位数。

**左图（蓝柱）**：各月均值，附带一倍标准差的误差棒。

**右图（橙柱）**：各月中位数，不受极端值影响。

**揭示的问题**：

- 中位数图（右）显示较清晰的季节规律：5 月和 6 月流量最高，7 月和 12 月最低，整体呈"春末秋初高、盛夏和冬季低"的双峰结构。
- 均值图（左）的误差棒极大（部分月份误差棒超过均值本身，甚至延伸到负值），说明各 bgrp 之间流量差异非常大，均值代表性差，中位数是更可靠的参考。
- 季节性幅度不算特别强，峰谷差约为均值的 30–40%，但足以在时序模型中引入偏差。
- **建模影响**：在特征工程中加入"日历月份"哑变量或正弦/余弦周期编码可以帮助模型捕捉这一规律。

## Figure 04 — STL 季节分解

**文件**：`figures/traffic_eda/04_stl_decomposition.png`

**生成数据**：将全市所有 bgrp 的月度流量加总后，形成一条单一的城市级月度时间序列（2010–2026），使用 STL（Seasonal-Trend decomposition via LOESS）方法进行分解，周期设为 12 个月。

**四个子图（由上至下）**：

- **Original**：原始城市级月度总流量，锯齿明显。
- **Trend**：长期趋势，2010–2016 年约在 60 万量级，2016 年后开始下滑，2020 年左右出现明显低谷（对应 COVID 冲击），此后有所回升但未恢复到历史高位，2025–2026 年数据较少导致趋势急速下跌。
- **Seasonal**：12 个月周期成分，振幅约 ±40 万，存在清晰的年度周期但各年振幅并不完全一致，说明季节性是时变的（time-varying seasonality）。
- **Residual**：残差，大多数月份接近 0，但存在若干尖峰（尤其在 2020 年前后），对应数据批量上报造成的异常月份。

**季节强度为 0.476**（0 = 无季节性，1 = 完美季节性），属于中等偏弱。

**揭示的问题**：

- 趋势的下滑不完全是真实交通量减少，也包含传感器覆盖率下降的影响（如 Figure 01 所示）。
- 残差中的尖峰是数据质量问题的直接体现，对下游建模是噪声。
- 季节强度适中，值得在模型中显式建模，但单独靠季节项无法解释大部分方差。
- **建模影响**：若使用 LSTM 或时序回归，建议将趋势项和季节项分别处理，而非直接输入原始值。

## Figure 05 — 空间分布地图

**文件**：`figures/traffic_eda/05_spatial_distribution_map.png`

**生成数据**：将面板中各 bgrp 的历年非零月均流量（年均平均）与 `data/processed/blocks.geojson` 的空间几何进行合并，按 `block_id` 匹配。

**左图（二值覆盖图）**：红色为有至少一个月非零记录的 block group，灰色为从未有记录的 block group。

**右图（流量大小图）**：仅展示有数据的 block group，颜色由浅黄到深红代表年均流量由低到高，灰色区域表示无传感器覆盖。

**揭示的问题**：

- 全市 28,790 个 block group 中仅 1,844 个（约 6.4%）有交通数据，传感器稀疏分布于主干道、高速路沿线，大部分居住区和支路没有覆盖。
- 高流量（深红色）集中在若干主要路段，如布鲁克林和皇后区的高速公路走廊，以及曼哈顿部分街道。
- 灰色区域（无数据）不等于"零流量"，而是"无传感器"，这是结构性缺失，与随机缺失有本质区别。
- **建模影响**：用 0 填充无覆盖区域会系统性低估这些区域的真实交通暴露。如需全面估计，需要引入空间插值（如克里金插值）或使用道路密度等代理变量来弥补传感器盲区。

## Figure 06 — 各月箱线图

**文件**：`figures/traffic_eda/06_boxplot_by_month.png`

**生成数据**：剔除零值后，按日历月份分组，分别绘制原始值和 log1p 变换后的箱线图。

**左图（蓝色）**：原始流量，纵轴单位为实际计数（0 至约 70 万）。

**右图（红色）**：log1p 变换后，纵轴单位为对数值（0 至约 14）。

**揭示的问题**：

- 左图中，每个月的箱体（IQR）压缩在 0 附近，而上方的须线和散点（异常值点）延伸非常长，说明数据高度右偏，大量极端值拉高了上界。中位数（橙线）在所有月份中都接近于零，印证多数有记录的 bgrp 流量较小。
- 右图（log1p 后），箱体明显展开，各月的中位数约在 8–10 之间，分布更加对称，离群点数量大幅减少，但仍存在少量低端异常值（接近 0 的 log1p 值，对应极小流量记录）。
- 从月间差异看，右图中 5 月和 10 月的箱体中位数略高，7 月和 12 月略低，与 Figure 03 结论一致。
- **建模影响**：对比两图可直观说明 log1p 变换的必要性，变换后分布更适合线性模型和梯度类模型。

## Figure 07 — 分布形态与 QQ Plot

**文件**：`figures/traffic_eda/07_distribution_qq.png`

**生成数据**：对面板中所有非零的 `traffic_volume_sum` 值，分别计算原始值和 log1p 值的分布，并与正态分布进行比较。

**四个子图**：

- **左上（蓝色直方图）**：原始流量分布。极度右偏，绝大多数值集中在 0–5 万区间，长尾延伸至约 70 万。偏度 = 4.40，峰度 = 26.20，远超正态分布（偏度 0，峰度 0）。
- **右上（红色直方图 + 黑色正态曲线）**：log1p 变换后，分布转变为左偏的单峰形态，偏度降至 −0.71，峰度降至 0.23，接近正态但仍有左尾偏移。黑色曲线是拟合的正态分布，可以看出主峰基本吻合，但左侧存在一个低值模式（log1p 值在 1–4 之间的小峰）。
- **左下（原始 QQ 图）**：数据点严重偏离对角线，尤其在高分位数区域向上大幅翘起，典型的重尾分布特征。
- **右下（log1p QQ 图）**：数据点在中间段（约 −2 到 +2 理论分位数）基本贴近对角线，但两端仍有偏离，说明 log1p 后左尾有轻微多余的小值，右尾有轻微重尾。

**揭示的问题**：

- 原始数据完全不适合假设正态性的统计方法（OLS、StandardScaler 等）直接使用。
- log1p 变换显著改善分布形态，但并不完美——左侧的双峰结构（低流量记录 vs 正常流量记录）提示数据中可能混有两种不同性质的来源（如短期临时测量 vs 长期固定传感器）。
- **建模影响**：推荐在建模前对 traffic 特征进行 log1p 变换，StandardScaler 应在变换后再应用。

## Figure 08 — 自相关分析（ACF / PACF）

**文件**：`figures/traffic_eda/08_acf_pacf.png`

**生成数据**：
- 全市级别：将所有 bgrp 的月均流量聚合为单一时间序列，取 log1p 后计算 ACF 和 PACF（最大滞后 36 个月）。
- bgrp 级别：选取数据最完整的单个 bgrp（`BX_5900`），对其 log1p 月度序列做相同分析。

**五个子图**：

- **左上（全市 ACF）**：lag-1 自相关约 0.25，lag-12 出现明显正峰（超出蓝色置信带），lag-24 处也有较弱正峰，符合年周期性特征。
- **右上（全市 PACF）**：lag-1 偏自相关约 0.25，lag-2 之后迅速衰减至置信带内，lag-6 和 lag-12 有轻微负偏自相关。
- **中左（时间序列）**：全市 log1p 月均流量序列，可见明显的锯齿波动叠加在较弱的长期趋势上，验证前面图表的观察。
- **中右（注释框）**：说明 lag-12 的 ACF 峰值对应年度季节性，PACF 快速衰减提示可能适合 MA 或季节性 MA 结构。
- **下两图（bgrp BX_5900 ACF/PACF）**：单一 bgrp 的自相关几乎全部在置信带内，说明单个 bgrp 的时间序列太短或太稀疏（中位数仅 1–2 个月数据），无法识别出有意义的自相关结构。

**揭示的问题**：

- 城市级聚合序列存在可识别的年周期性（lag-12 显著），但单个 bgrp 级别的时序信号非常微弱，原因是大多数 bgrp 缺乏足够长的连续观测。
- 这意味着时序建模更适合在聚合层面（全市或区域级）进行，而不是对每个 bgrp 单独建时序模型。
- **建模影响**：对于面板模型，建议将"日历月份"和"年份"作为显式特征输入，而非依赖模型自动捕捉滞后结构。若使用 LSTM，需要注意大多数 bgrp 的有效序列长度不足以支持长程依赖学习。

## Figure 09 — 极端值处理对比

**文件**：`figures/traffic_eda/09_extreme_values.png`

**生成数据**：对全量非零记录，分别展示四种处理方式下的分布形态。

**四个子图**：

- **左上（原始值，蓝色）**：极度右偏，p99 = 336,055（红色虚线），绝大多数值堆积在 0–5 万区间，长尾延伸至约 70 万。
- **右上（log1p，红色）**：变换后分布转为接近正态的单峰，主峰在 log1p 值约 9–10 之间（对应实际值约 8,000–22,000），分布集中且较对称。
- **左下（Winsorize @ p99，绿色）**：将超过 336,055 的值压缩至 336,055，分布形态与原始相近，但右尾被截断，最右侧出现一个尖峰（被压缩的极值堆叠点）。整体仍然严重右偏，改善有限。
- **右下（Clip @ p99.9，紫色）**：截断阈值更宽松（638,104），仅去除最极端的 0.1%，分布形态与原始几乎相同，说明极值非常密集，宽松的截断效果不明显。

**关键数值**：p95 = 167,630，p99 = 336,055，p99.9 = 638,104。超过 p99 的记录占有效记录的约 1%。

**揭示的问题**：

- 对比四种方法，**log1p 变换**是唯一能真正改变分布形态的处理方式，winsorize 和 clip 在原始尺度上改善有限。
- 建议的处理流程：先 log1p 变换，再对变换后的值做 winsorize（去除对数空间下的极端值），最后 StandardScaler 标准化，这样三者结合效果最好。
- **建模影响**：如果直接将原始值输入 StandardScaler，少数极端值会导致绝大多数正常值被压缩至接近 0 的区间，严重损害模型性能。

## Figure 10 — bgrp 间空间异质性

**文件**：`figures/traffic_eda/10_bgrp_distribution.png`

**生成数据**：剔除零值后，按 bgrp 计算每个 block group 的历史中位数流量，然后分析这些中位数的分布和排名。

**左图（排序曲线）**：横轴为 bgrp 排名（从低流量到高流量），纵轴为各 bgrp 的流量中位数。曲线在约前 1,200 名时几乎贴近 0，从 1,200 名开始缓慢上升，1,500 名后急速拉升，最高的少数 bgrp 中位数超过 60 万，形成典型的幂律分布形态。

**右图（直方图）**：大多数 bgrp 的中位数集中在 0–5 万区间，均值（25,146）远大于中位数（9,424），分布强烈右偏。

**揭示的问题**：

- 各 bgrp 之间的交通量差异极大，少数 block group（大概率是靠近主干道/高速路的区域）承担了绝大部分流量，而大多数 block group 流量极低。
- 均值（25,146）是中位数（9,424）的约 2.7 倍，说明高流量的少数 bgrp 将均值大幅拉高，使用均值作为代表值会显著高估"典型"区域的流量水平。
- 空间异质性极高，这意味着 traffic 变量必须在 bgrp 级别保留，不能用全市均值代替，否则会丢失大量空间信息。
- **建模影响**：在面板模型中需引入 bgrp 固定效应或 bgrp 级别的静态特征（如道路密度、与主干道的距离）来控制这种空间异质性，避免残差存在系统性的空间自相关。

## 综合结论

| 问题 | 关键发现 | 建模建议 |
|------|----------|----------|
| **数据覆盖** | 仅 6.4% 的 bgrp 有记录，且多数只有 1–2 个月 | 区分"无传感器"和"零流量"；不宜对无覆盖区域直接填 0 |
| **时间规律** | 城市级月度序列高度不稳定，受数据上报批次影响 | 使用传感器数量归一化；建议二值化或用覆盖率替代绝对值 |
| **季节性** | 年周期性存在（STL 季节强度 0.476），5–6 月高，7 月和 12 月低 | 加入月份哑变量或正弦/余弦编码 |
| **分布形态** | 原始值极度右偏（偏度 4.40），log1p 后接近正态（偏度 −0.71） | 建模前必须做 log1p 变换 |
| **极端值** | p99 = 336,055，约 1% 记录超过 p99 | log1p 后再 winsorize，最后 StandardScaler |
| **空间异质性** | 各 bgrp 中位数从接近 0 到 60 万以上，差异极大 | 保留 bgrp 固定效应或引入道路密度等空间协变量 |
| **自相关** | 城市级 lag-12 显著，单 bgrp 序列太短无法识别 | 对单 bgrp 使用月份特征，不建议对单 bgrp 建时序模型 |
